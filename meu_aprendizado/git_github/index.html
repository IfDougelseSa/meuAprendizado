Git e Github

Essa página foi feita para o estudo/revisão de Git e GitHub.


O git é um sistema de gerenciamento de controle de versão de código/documentos, o mesmo possui recursos avançados de monitoramento e acompanhamento de arquivos que fazem parte de um projeto. Exemplo: Ao criar um projeto e adiciona-lo no git, é possivel monitora-lo e fazer alterações sem prejudicar o processo principal.

O ciclo de vida do status de arquivos do Git

Untracked

Quando iniciamos o comando git init para iniciar o git em um novo projeto, todos os arquivos desse mesmo projeto estarão no estado untracked. Isso significa que, apesar do arquivo fazer parte do projeto ele ainda não foi adicionado ao monitoramento do git, ou seja, ele não está "rastreado".

Quando não houver monitoramento do arquivo o git não estará controlando o versionamento desse arquivo.


Tracked

Tracked: Quando o arquivo é adicionado no monitoramento do Git, o estado desse arquivo passa a ser 'tracked', ou seja, rastreado. A partir desse estágio que todo poder do git comeca, pois ele passa a controlar as mudancas do arquivos. Além disso, os arquivos do estado tracked são conhecido como new file.

Modified

Modified: Quando o arquivo que está sendo rastreado sofre mudanças ele passa a ser modified.

Staged

Quando um arquivo já foi modificado e finalizado, ele passa para o estágio staged, sendo enviado para o repositório através de commits.

Os primeiros passos no git

1 -git init - untracked

Para adicionar o git ao seu projeto use o comando git init, porém os arquivos do projeto estarão untracked, ou seja, não monitorados.

2- gid add "nome_do_arquivo" - tracked

Os arquivos estão sendo monitorados.

Alterando o arquivo - modified

Suponha que você tenha alterado um arquivo tracked, então seu arquivo estará no estado modified.

3- git add - tracked

Para deixar seu arquivo tracked novamente, apenas use o git add "nome do arquivo". Pronto! O arquivo está no estado tracked.

4- git commit -m "mensagem_relevante_sobre_o_arquivo" - staged

Por último e não menos importante, toda vez que mudanças relevantes forem feitas no arquivo, adicionar um commit sempre será bom, pois saberá o que foi mudado e quando.
